# 🚀 古典占星程式：GitHub 雲端更新教學手冊

這份手冊是為了讓你即使不懂程式設計，也能輕鬆管理你的數位資產。每當你在電腦上修改了程式內容（例如改文字、換地名標籤），只要按照以下步驟操作，網頁就會自動更新。

## 第一步：開啟終端機並「定位」資料夾

因為資料夾已經搬家，我們使用最直覺的拖曳法來告訴電腦資料夾的新位置：

1. 打開你的 **終端機 (Terminal)**。
2. 輸入 `cd ` （注意：**cd 後面必須有一個空格**）。
3. 用滑鼠按住你的 `my-astro-app` 資料夾，直接**拖進**終端機視窗裡，電腦會自動填好路徑。
4. 按下 **Enter** 鍵。

---

## 第二步：執行「發射三部曲」指令

當終端機已經進入資料夾後，請依序執行這三行核心指令。這就像是把修改好的包裹打包、貼標籤、最後寄出的過程。

### 1. 打包修改內容 (git add)

這行指令會把你在資料夾內所有做過的變動通通抓起來放進準備區。

> `git add .`

### 2. 貼上更動標籤 (git commit)

這行指令是為了記錄你這次做了什麼。引號內的文字你可以自訂。

> `git commit -m "更換地圖 API 為 ArcGIS 以修復連線錯誤"`

### 3. 正式發射到雲端 (git push)

這是最後一步，會把剛才打包好的內容正式推送到 GitHub 網頁上。

> `git push`

---

## 第三步：檢查網頁是否更新

執行完上述三行指令後，請稍等約 10 到 20 秒：

1. 開啟你的排盤網址：`https://easyastrology.streamlit.app/`
2. **重新整理網頁**。
3. 確認修改內容是否已經生效。

## 💡 技術小撇步：為什麼這樣做？

* **`cd` (Change Directory)**：就像是走路進去你的工作室。
* **`git add .`**：像是把桌上修好的零件通通裝進箱子裡。
* **`git commit`**：在箱子外面寫上備註，方便以後查閱。
* **`git push`**：就像是把快遞寄出去，雲端伺服器收到後會自動幫你換上新的零件。

這套流程能讓你穩定地經營馬來西亞與台灣的命理事業，讓你在外送工作的空檔也能隨時透過手機維護你的專業工具。

優化後的標準流程：

cd [拖入資料夾]

git pull origin main（先跟雲端打招呼，看看有沒有新東西）

git add .

git commit -m "更新內容"

git push
